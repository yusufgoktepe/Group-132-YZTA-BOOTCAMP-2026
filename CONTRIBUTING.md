# Contributing

## Branch Kullanımı

- `main`: Kararlı sürüm
- `feature/...`: Yeni özellikler
- `docs/...`: Dokümantasyon güncellemeleri
- `fix/...`: Hata düzeltmeleri

## Çalışma Akışı

1. Issue seçilir.
2. Branch oluşturulur.
3. Değişiklik yapılır.
4. Pull Request açılır.
5. Takım kontrolünden sonra merge edilir.

## Scrum Notu

Sprint sürecindeki önemli kararlar ilgili `scrum/sprint-x/` klasörüne eklenmelidir.
